# YAML Data CMS Editing — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the four YAML data collections (CV, Stack, Labs, PhD Progress) editable through the existing Sveltia CMS.

**Architecture:** Restructure YAML files from bare arrays to keyed maps (required by Sveltia's "file" collection type), update Astro's content config to drop the now-implicit `id` field, add four file collections to the CMS config, and fix the one page that references `data.id`.

**Tech Stack:** Astro 5, Sveltia CMS, YAML, Zod

### Task 2: Update Astro content config to drop `id` from YAML schemas

**Files:**
- Modify: `src/content.config.ts:44-89` (the four YAML collection definitions)

- [ ] **Step 1: Remove `id: z.string()` from all four YAML collection schemas**

In `src/content.config.ts`, update the four YAML collection definitions. Remove the `id: z.string(),` line from each schema object. The schemas should become:

```typescript
// ── YAML data collections (file) ──

const cv = defineCollection({
  loader: file('./src/data/cv.yaml'),
  schema: z.object({
    section: z.enum(['education', 'clinical', 'research', 'skills']),
    year: z.string(),
    title: z.string(),
    institution: z.string(),
    description: z.string(),
  }),
});

const stack = defineCollection({
  loader: file('./src/data/stack.yaml'),
  schema: z.object({
    layer: z.string(),
    methods: z.string(),
    tools: z.string(),
    description: z.string(),
    order: z.number(),
  }),
});

const labs = defineCollection({
  loader: file('./src/data/labs.yaml'),
  schema: z.object({
    name: z.string(),
    pi: z.string(),
    institution: z.string(),
    role: z.string(),
    work: z.string(),
    url: z.string().optional(),
  }),
});

const phdProgress = defineCollection({
  loader: file('./src/data/phd-progress.yaml'),
  schema: z.object({
    label: z.string(),
    value: z.number(),
    color: z.string().optional(),
  }),
});
```

- [ ] **Step 2: Commit**

```bash
git add src/content.config.ts
git commit -m "refactor: remove id field from YAML collection schemas"
```

### Task 5: Add YAML file collections to Sveltia CMS config

**Files:**
- Modify: `public/admin/config.yml`

- [ ] **Step 1: Add four file collections to the `collections` list**

Append the following to the end of the `collections` list in `public/admin/config.yml`, after the existing `misc` collection:

```yaml

  # ── CV (public, data) ─────────────────────────────

  - name: cv
    label: "CV"
    file: src/data/cv.yaml
    fields:
      - { label: Section, name: section, widget: select, options: [education, clinical, research, skills] }
      - { label: Year, name: year, widget: string }
      - { label: Title, name: title, widget: string }
      - { label: Institution, name: institution, widget: string }
      - { label: Description, name: description, widget: text }

  # ── Stack (public, data) ──────────────────────────

  - name: stack
    label: "Stack"
    file: src/data/stack.yaml
    fields:
      - { label: Layer, name: layer, widget: string }
      - { label: Methods, name: methods, widget: string }
      - { label: Tools, name: tools, widget: string }
      - { label: Description, name: description, widget: text }
      - { label: Order, name: order, widget: number, value_type: int }

  # ── Labs (public, data) ───────────────────────────

  - name: labs
    label: "Labs"
    file: src/data/labs.yaml
    fields:
      - { label: Name, name: name, widget: string }
      - { label: PI, name: pi, widget: string }
      - { label: Institution, name: institution, widget: string }
      - { label: Role, name: role, widget: string }
      - { label: Work, name: work, widget: text }
      - { label: URL, name: url, widget: string, required: false }

  # ── PhD Progress (public, data) ───────────────────

  - name: phd-progress
    label: "PhD Progress"
    file: src/data/phd-progress.yaml
    fields:
      - { label: Label, name: label, widget: string }
      - { label: Value, name: value, widget: number, value_type: int, min: 0, max: 100 }
      - { label: Color, name: color, widget: color, required: false }
```
